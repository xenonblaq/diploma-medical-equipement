# Остаток экспорта и импорта.

Данный Jupyter Notebook предназначен для **загрузки, подготовки, анализа и прогнозирования остатка экспорта и импорта на уровне страна+HS4**  

---

## Используемый стек

### Основные библиотеки
- `pandas`, `numpy` — работа с данными
- `matplotlib`, `seaborn` — визуализация
- `requests`, `xml.etree.ElementTree` — загрузка данных ЦБ РФ (курс USD/RUB)
- `duckdb` — хранение и агрегация данных
- `statsmodels` — STL‑декомпозиция
- `pmdarima` — ARIMA‑модели
- `sktime` — метрики и разбиение временных рядов
- `catboost` — ML‑модель для регрессии
- `tqdm` — прогресс‑бар

## Логика работы ноутбука (по шагам)

### 1. Импорт библиотек и настройка окружения
В первых ячейках:
- подключаются все зависимости,
- отключаются warning‑сообщения,
- настраивается отображение графиков.

Цель этапа — подготовить чистое рабочее окружение.

---

### 2. Сбор временных рядов импорта, зашумление

Собираем ряды импорта по странам и товарам (4-ем знакам) с 2019 по июль 2025-ого года из различных файлов.
Приводим к общей структуре, оставляем ряды, доступные во всех датасетах - пересечение страна+код по всем годам.

С помощью CatBoost переобучаем модель и таким образом зашумляем данные (в чистом виде данные нельзя использовать из-за конфиденциальности).

---

### 3. Сбор временных рядов экспорта, прогнозирование

Собираем ряды экспорта по странам и товарам (4-ем знакам) с 2019 по 2024 год из различных файлов.
Приводим к общей структуре, оставляем ряды, доступные во всех датасетах - пересечение страна+код по всем годам.

Прогнозируем значения каждого ряда до выбранной даты с помощью модели SARIMA.

---

### 4. Выгрузка в БД

Итоговые данные экспорта после зашумления и импорта после прогнозирования сохраняются в excel файлы. Далее с помощью специальной функции `prepare` приводим данные к формату БД и записываем по WRITE-токену.
